                              # 本地缓存 VS 分布式缓存

                              本地缓存和分布式缓存在架构和使用场景上存在显著区别：

                              本地缓存分布式缓存
                              含义与应用程序在同一台主机节点上，多是进程间访问，与 JVM 生命周期一致，大多不会产生网络 IO，因此访问速度更快大多具备独立部署的主机节点，跨进程访问，会产生网络 IO，访问速度较本地缓存更慢，支持的数据结构更多
                              持久性大多不具备持久性，存在的生命周期与 JVM 的生命周期一致根据分布式缓存的技术选型而定，如 redis 具有完善的持久化机制
                              存储空间受 JVM 堆空间大小限制受部署节点的物理主机的内存大小限制
                              技术选型实现简单，在项目中集成相关的组件即可，如 ehcache、guava 等实现也较为简单，如 Redis、Memercached 等
                              适用场景适用于读多写少的，变更较少的业务场景，比如数据字典等支持更多的业务场景
                              一致性- 多个应用实例之间无法共享数据，一致性较差；- 与数据库数据存在一致性问题，需在应用端进行保证；- 可在多个应用实例之间共享数据，一致性较好；- 仍然存在与数据库数据不一致的情况，也需要在应用端进行保证；
